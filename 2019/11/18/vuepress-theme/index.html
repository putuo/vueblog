# 如何开发 vuepress 主题

我平时主要用的是 angular，偶尔也会用 vue 开发，但不多 , 现有个任务开发出一套 vuepress 自定义主题风格并上传到 github，最近使用vue都在几个月之前了，时间紧迫有点吃力，本文就在此情形下进行开发。

这里我分享下我开发的思路，有的时候技术获取不难，但是想法却很重要，希望我这一点点的经验能够读者带来些帮助。

vuepress 安装是必须，概述如下：

yarn create vuepress
// 选择 blog 一路 enter
yarn install 
// 安装依赖
yarn dev
// 打开 localhost 看下

站点跑起来了 🙂

官网文档很清晰，但若详细看下来，可费不少时间，那么我们能否在他人的技术开发呢？找了一圈，好尴尬没找到社区，零星有些又貌似太过于强大，不适合，那么我们从官方入手吧，看了下 .vuepress/config.js 配置文件

module.exports = {
    theme: '@vuepress/theme-blog',    
}

嗯，在它基础上修改是否会好些呢？

找到 node_modules\@vuepress\theme-blog ，同目录下直接另存一份为 theme-blog-edit ，修改 .vuepress/config.js 配置文件

module.exports = {
    theme: '@vuepress/theme-blog-edit',    
}

yarn dev
// 正常

不错，离目标近些。

接下来就要定位下自己想开发成什么样的博客，推荐一个站点：https://themeforest.net/ 里面有很多漂亮的模板，大部分是要收费的，当然了，看看又不要钱，最后定位到了一个包含时间轴主题博客：

hoho，应该就是它了，盘它。

# 文章内容

接下来就是列表和文章了，考虑到列表有分页操作，先做文章模块。

文章模块如果不看文档可能会有些懵，那么文档还是要看下的，

获取渲染内容

当前的 .md 文件渲染的内容，可以作为一个独特的全局组件 <Content/> 来使用，你可能想要它显示在页面中的某个地方。一个最简单的主题，可以是一个唯一的 Layout.vue 组件

它就是文章的内容，ok安心使用，但突然有个想法文章标题能够单独拿出来用么，不然一大篇文章，自己去截取？有看了下文档

Layout 组件将会对每一个文档目录下的 .md 执行一次，同时，整个网站以及特定页面的元数据将分别暴露为 this.$site 和 this.$page 属性，它们将会被注入到每一个当前应用的组件中。

我们可以在当前页面中使用 this.$page 就可以拿到所有数据了，这个时候我对 Content 进行了改造，新建一个组件 ArticlePage

# 列表页模块

剖析下 BaseListLayout 这个组件应该很容易进行排版

现在问题在于如何分页，固然使用原先的分页是可以了，但是和原样式相差甚远，最好是滚动加载出来，不过~，先实现点击 More 加载更多。

当前主题就包含了分页的功能，能否直接使用呢？

先把 $pagination 输出出来看下，呵呵东西不少，但貌似没有啥可直接调用的方法，几乎都是些属性，没办法如果不自己开发组件的话，那就先研究下它的源码，经过 pageination 的全局搜索，最终定位到这个文件：node_modules\@vuepress\plugin-blog\lib\client\pagination.js

pages 方法可以根据 this._currentPage.interval 拿到当前分页数据，好吧，暂时这样了：
